## 1. マクロ・VBAの解説

### 1-1. マクロとVBAとは

- **マクロ**: Excel上で行う手作業（コピー、書式設定、集計、グラフ作成…）を、
  あらかじめ手順書として書いておき、自動で実行させる仕組みのことです。
- **VBA（Visual Basic for Applications）**: その手順書を書くための言語です。

つまり「マクロ＝Excelを自動で動かす手順書」、「VBA＝その手順書を書くための言葉」という関係です。
プログラミングとは、**普段人間が手で繰り返している作業を、機械にわかる言葉で指示すること**だと
考えてください。毎月同じ手順で行っている集計作業ほど、マクロ化の効果が大きくなります。

### 1-2. 準備（開発タブの表示とファイルの保存形式）

**開発タブの表示**: マクロの機能は既定では表示されていません。
「ファイル」→「オプション」→「リボンのユーザー設定」で、右側の一覧の「開発」に
チェックを入れてOKを押すと、リボンに「開発」タブが表示されます。

**保存形式に注意**: マクロを含むブックは、通常の`.xlsx`形式では保存できません
（保存してもマクロが消えます）。「名前を付けて保存」で、ファイルの種類を
**「Excel マクロ有効ブック (\*.xlsm)」** に変更して保存してください。
また、`.xlsm`ファイルを開き直したときに黄色い警告バーが出た場合は、
「コンテンツの有効化」をクリックしないとマクロは動きません。

### 1-3. VBAのコードを書く場所と実行方法

1. 「開発」タブ→「Visual Basic」をクリックすると、VBAエディタ（別ウィンドウ）が開きます
2. エディタのメニューで「挿入」→「標準モジュール」を選ぶと、コードを書く白紙の画面が開きます
3. コードは`Sub`（サブプロシージャ）という単位で書きます。これが「マクロ1つ」にあたります

```vba
Sub 練習()
    ' ここに処理を書く（行頭の「'」から後ろはコメントで、実行されません）
End Sub
```

`Sub 練習()` と入力してEnterキーを押すと、`End Sub` は自動で補完されます。
書いたコードは、**そのSubの中にカーソルを置いた状態でF5キー**を押すと実行できます。
Excel側の「開発」タブ→「マクロ」からマクロ名を選んで実行することもできます。

`MsgBox 値` と書くと、画面にメッセージウィンドウで値を表示できます。
コードが意図通り動いているかを確認するのに便利なので、演習中はどんどん使ってください。

### 1-4. 変数・条件分岐・繰り返し

**変数**: 値を一時的に入れておく「箱」です。使う前に`Dim 名前 As 型`で宣言します。

```vba
Dim x As Integer   ' 整数を入れる箱
x = 10             ' 箱に10を入れる
```

| 型 | 用途 |
|------|-----------|
| `Integer` | 小さい整数 |
| `Long` | 大きい整数（行番号にはこちらを使う） |
| `Double` | 小数を含む数値（平均値などに使う） |
| `String` | 文字列 |

**条件分岐（If）**: 条件が成り立つときだけ処理を実行します。

```vba
If x > 5 Then
    MsgBox "5より大きい"
Else
    MsgBox "5以下"
End If
```

**繰り返し（For）**: 決まった回数だけ同じ処理を繰り返します。
`i`は「今何回目か」を表す変数で、1回ごとに自動で1ずつ増えます。

```vba
Dim i As Long
For i = 1 To 5
    MsgBox i        ' 1,2,3,4,5 と順に表示される
Next i
```

この`For`の`i`を「行番号」として使うことで、**表を1行ずつ上から処理していく**という、
実務のマクロで最もよく使う形になります。

### 1-5. セル・シートの操作（Range / Cells / Sheets）

VBAからセルを指定する方法は主に2つあります。

| 書き方 | 意味 |
|--------|------|
| `Range("A1")` | A1セル。人が読む「A1」形式で指定する |
| `Cells(2, 1)` | 2行1列目（＝A2セル）。**行番号・列番号を数値で指定する** |

`Cells`は行・列を数値で指定できるため、`For`の変数`i`と組み合わせて
`Cells(i, 1)`のように書けます。**1行ずつ処理するループでは`Cells`を使う**と覚えてください。

```vba
Range("A1").Value = "テスト"           ' A1に文字を入れる
Cells(2, 1).Value = 100                ' A2に100を入れる
Sheets("Sheet1").Range("B1").Value = "シート指定"   ' シート名を明示して指定
```

`.Value`はそのセルの「値」を表します。代入すれば書き込み、`x = Range("A1").Value`のように
書けば読み取りになります。

### 1-6. Excelの関数をVBAから呼び出す（WorksheetFunction）

SUM・AVERAGE・COUNTIFなど、シート上で使っている関数は、VBAからも呼び出せます。

```vba
Dim total As Double
total = WorksheetFunction.Sum(Range("A1:A5"))     ' A1:A5の合計
MsgBox total
```

同じように`WorksheetFunction.Average(...)`、`WorksheetFunction.SumIf(...)`、
`WorksheetFunction.CountIf(...)` も使えます。集計処理を自分でループを書いて計算する
こともできますが、既存の関数を呼ぶほうが短く・間違いが少なくなります。

### 1-7. エラー処理の基本

存在しないシートを参照するなど、実行できない命令に出会うとマクロは途中で止まり、
エラーメッセージが表示されます。これを制御するのがエラー処理です。

```vba
On Error Resume Next    ' 以降、エラーが起きても止まらず次の行へ進む
Sheets("存在しないシート名").Select
On Error GoTo 0         ' エラー処理を解除し、通常どおりエラーで止まる状態に戻す
```

`On Error Resume Next`は便利ですが、**すべてのエラーを黙って無視してしまう**ため、
必要な範囲だけに使い、`On Error GoTo 0`で必ず解除するのが基本です。

### 1-8. データ範囲の相対取得（今回の最重要ポイント）

セル範囲を`Range("A2:A16")`のように**固定値**で書くと、データの件数が変わった瞬間に
マクロが正しく動かなくなります（データが増えた分を処理し損ねる／減ると空セルを処理して
エラーになる）。実務で扱うデータは毎回件数が変わるため、
**「今、何行分データが入っているか」をコードの中で調べてから処理する**書き方が必須です。

```vba
Dim lastRow As Long
lastRow = Cells(Rows.Count, 1).End(xlUp).Row   ' A列でデータが入っている最終行の行番号
```

これは「A列の一番下のセルから上に向かってCtrl+↑を押す」という操作をコードにしたものです。
求めた`lastRow`をループの終了条件に使えば、件数が変わっても書き換え不要になります。

```vba
For i = 2 To lastRow    ' 2行目（見出しの次）から最終行まで
    ' 1行ずつの処理
Next i
```

同様に`Range("A1").CurrentRegion`は、A1を含む「連続したデータの塊」全体を表します
（`.Rows.Count`で行数、`.Columns.Count`で列数が取れます）。

## 2. ステップ演習1: マクロの記録と手動編集（目安50分）

> **なぜ「記録」から始めるか**: マクロ記録機能そのものは実務ではほとんど使いません
> （生成されるコードは冗長で非効率なため、実務では直接VBAを書きます）。
> ここでは「操作する → 生成されたコードを見る」という体験を通じてVBAの文法に慣れることと、
> **やり方の分からない操作のコードを調べる手段**として使えることを知るのが目的です。

1. 「ファイル」→「オプション」→「リボンのユーザー設定」で「開発」にチェックを入れ、
   開発タブを表示する
2. 適当な表（3行×3列程度でよい）をシートに用意する
3. 「開発」タブ→「マクロの記録」をクリックし、マクロ名を入力してOKを押す
4. 表のセル範囲を選択し、罫線を引く操作を行う
5. 「開発」タブ→「記録終了」をクリックする
6. 「開発」タブ→「Visual Basic」をクリックし、生成されたコードを確認する
   → 自分が行った操作が`Range(...)`や`Borders(...)`といったコードになっていることを確認する
7. コード内のセル範囲（例: `Range("A1:C3")`）を、別の範囲（例: `Range("A5:C7")`）に書き換える
8. Excelに戻り、「開発」タブ→「マクロ」から実行し、書き換えた範囲に罫線が引かれることを確認する

## 9. Day7 応用課題（自力で取り組んでください・目安6〜8時間）

ここからは手順の指示はありません。1〜8章で習得した要素を組み合わせて、自分で考えて
取り組んでください。

### シナリオ

社内アンケートの回答データが届きました。**部署別の満足度平均を集計し、グラフ化するまでを
ボタン1つで実行できるマクロ**を作成してください。アンケートは今後も繰り返し実施され、
そのたびに回答件数が変わります。

使用ファイル:
- `day7_survey_sample.csv`（回答15件。回答者ID・部署・満足度(5段階)・自由記述有無）
- `day7_survey_sample_2.csv`（回答21件。列の構成は同じで、件数と内容だけが異なる）

### 必須条件

**行数の異なる2つのデータのどちらに対しても、コードを一切書き換えずに同じマクロが
正常に動くこと。** セル範囲や行数を固定値で書いてはいけません（8章の相対取得を使うこと）。
ここが今回の課題の中心です。

### 課題

1. `day7_survey_sample.csv` の内容をシートに取り込む
2. 部署別（営業部・開発部・総務部）の満足度平均を求めるマクロを作成する
   （Forループで1行ずつ判定して集計する方法でも、`WorksheetFunction.AverageIf`等の
   関数を使う方法でも、どちらでも構いません）
3. データ範囲は `Cells(Rows.Count, 1).End(xlUp).Row` 等を使って相対的に取得し、
   件数が変わっても書き換え不要にする
4. 集計結果をもとに、グラフ（棒グラフ等）を自動生成する処理をマクロに追加する
5. 1〜4の処理を、1つのボタンで連続実行できるようにまとめる
6. 処理のどこかにエラー処理（`On Error Resume Next` 等）を組み込む
7. `day7_survey_sample.csv` で実行して結果を確認した後、シートのデータを
   `day7_survey_sample_2.csv` の内容に入れ替え、**同じマクロ・同じボタンで再実行**して、
   コードを一切変更せずに正しい結果が出ることを確認する

> **グラフ作成のヒント**: グラフをVBAで作る方法はまだ解説していません。
> こういうときこそ、ステップ演習1で学んだ**マクロの記録**が使えます。
> 手作業でグラフを挿入する操作を記録し、生成されたコードを読めば、
> `ActiveSheet.Shapes.AddChart2 ...` や `.SetSourceData Source:=Range(...)` といった
> 書き方が分かります。それを自分のマクロに組み込み、不要な行を削って整えてください。
> なお、実行のたびにグラフが増えていかないよう、作成前に既存のグラフを削除する
> （`ActiveSheet.ChartObjects.Delete`）処理を入れておくとよいでしょう。

### 確認用の答え合わせ（部署別の満足度平均）

| 部署 | `day7_survey_sample.csv`（15件） | `day7_survey_sample_2.csv`（21件） |

### 発展課題（時間が余った場合）
