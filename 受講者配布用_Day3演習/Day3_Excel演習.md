## 1. 関数・機能の解説

### 1-1. MATCH関数・INDEX関数とは

**MATCH関数**: 指定した値が範囲の中で何番目にあるか（位置）を返す関数です。

```
=MATCH(検索値, 検索範囲, 照合の種類)
```

照合の種類は`0`（完全一致）を基本的に使います。

**INDEX関数**: 範囲の中から「上から◯番目・左から◯番目」の値を取り出す関数です。

```
=INDEX(範囲, 行番号, 列番号)
```

この2つを組み合わせると、「行の位置」と「列の位置」をそれぞれMATCHで求めて、
INDEXに渡すことで**表の行×列が交差するセルの値**を取り出せます。

```
=INDEX(値の範囲, MATCH(行の検索値, 行見出しの範囲, 0), MATCH(列の検索値, 列見出しの範囲, 0))
```

VLOOKUP/XLOOKUPは「1つの列（またはキー）から値を引く」のが得意ですが、INDEX+MATCHは
「商品×サイズ」のように**2つの条件の組み合わせで1つの値を特定したいとき**に使います。

### 1-2. 論理関数（IFS・AND・OR・IFERROR）

| 関数 | できること |
|------|-----------|
| `IFS(条件1,値1,条件2,値2,...)` | IFを何重にもネストせずに、複数の条件を上から順に判定できる |
| `AND(条件1,条件2,...)` | すべての条件が真のときだけTRUEになる |
| `OR(条件1,条件2,...)` | いずれかの条件が真であればTRUEになる |
| `IFERROR(数式, エラー時の値)` | 数式がエラーになったとき、代わりに指定した値を表示する |

IFSは最後の条件に`TRUE`を指定すると、「それ以外すべて」という受け皿として使えます。

```
=IFS(条件1, 値1, 条件2, 値2, TRUE, それ以外の値)
```

### 1-3. 名前付き範囲

セル範囲に「名前」を付けておくと、数式の中で`$B$2:$D$4`のようなセル番地の代わりに、
`価格表`のような分かりやすい名前で参照できます。数式の可読性が上がり、範囲がずれる
心配も減ります。名前を付けるには、範囲を選択して数式バー左の「名前ボックス」に
名前を入力してEnterキーを押します。

### 1-4. データ入力規則

セルに入力できる値の種類を制限する機能です。誤入力の防止に使います。

- リストからの選択のみ許可する（例: 名前付き範囲をリストの元データに指定する）
- カスタム数式で条件（整数のみ、一定範囲内のみ等）を指定する

## 2. ステップ演習1: INDEX＋MATCH関数（2軸検索・目安50分）

使用ファイル: `day3_product_price_table.csv`（商品価格表: 行=商品名、列=S/M/L）、
`day3_order_data.csv`（受注データ: 受注ID・商品名・サイズ・数量・特急対応・在庫数）

1. Excelを開き、新しい空白のブックを作成する
2. シート名を「価格表」に変更し、`day3_product_price_table.csv`の内容をA1セルに貼り付ける
   （A列=商品名、B1:D1=S/M/L、B2:D4が価格のマス目になる）
3. 新しいシートを追加し、シート名を「受注データ」に変更し、`day3_order_data.csv`の内容を
   A1セルに貼り付ける（A列=受注ID、B列=商品名、C列=サイズ、D列=数量、E列=特急対応、F列=在庫数）
4. 「受注データ」シートのG1に「単価」と入力する
5. G2セルに次の数式を入力する
   ```
   =INDEX(価格表!$B$2:$D$4,MATCH(B2,価格表!$A$2:$A$4,0),MATCH(C2,価格表!$B$1:$D$1,0))
   ```
   → B2（商品名）が「Tシャツ」、C2（サイズ）が「M」であれば「1600」が表示されればOK
6. 数式の意味を確認する
   - `MATCH(B2,価格表!$A$2:$A$4,0)` → 価格表の中で商品名が何行目にあるか
   - `MATCH(C2,価格表!$B$1:$D$1,0)` → 価格表の中でサイズが何列目にあるか
   - `INDEX(価格表!$B$2:$D$4, 行番号, 列番号)` → その行×列の交点の値を取り出す
7. G2をコピーし、最終行（G14）まで貼り付ける
8. G14（受注ID: O013、商品名: パンツ）を確認する → 価格表に「パンツ」が存在しないため
   `#N/A`エラーが表示されることを確認する（ステップ演習2でこのエラーに対処します）

### 確認用の答え合わせ（一部抜粋）

| 受注ID | 商品名 | サイズ | 単価 |
|--------|--------|--------|------|
| O001 | Tシャツ | M | 1600 |
| O002 | パーカー | L | 3600 |
| O003 | キャップ | S | 1200 |

## 3. ステップ演習2: 論理関数の組み合わせ（IFS・AND・IFERROR・目安50分）

使用ファイル: ステップ演習1で作成した「受注データ」シート（G列まで入力済みの状態から続ける）

1. G2セルの数式を、IFERRORで包んで書き直す（G2〜G14全体に再度コピーする）
   ```
   =IFERROR(INDEX(価格表!$B$2:$D$4,MATCH(B2,価格表!$A$2:$A$4,0),MATCH(C2,価格表!$B$1:$D$1,0)),"商品コード確認")
   ```
   → G14（パンツ）が`#N/A`ではなく「商品コード確認」という分かりやすい表示に変わることを確認する
2. H1セルに「割引率」と入力する。H2セルに、数量（D列）に応じた割引率をIFSで判定する数式を入力する
   ```
   =IFS(D2>=30,0.1,D2>=10,0.05,D2<10,0)
   ```
   → 数量30以上は10%、10〜29は5%、9以下は0%になることを確認する
3. H2をコピーし、H14まで貼り付ける
4. I1セルに「小計」と入力する。I2セルに、単価×数量×(1-割引率)を計算する数式を入力する
   ```
   =IFERROR(G2*D2*(1-H2),"")
   ```
   （単価が「商品コード確認」の文字列になっている行は掛け算がエラーになるため、
   IFERRORでエラー時は空欄にしている）I2:I14に貼り付ける
5. J1セルに「要確認」と入力する。J2セルに、「特急対応が『有』、かつ在庫数が数量より
   少ない」場合にだけ「要確認」と表示する数式を入力する
   ```
   =IF(AND(E2="有",F2<D2),"要確認","")
   ```
   → E列が「有」でF列（在庫数）がD列（数量）を下回る行だけ「要確認」と表示されることを確認する
6. J2をコピーし、J14まで貼り付ける。受注ID O002・O004・O009の3件が「要確認」に
   なっていれば正しい（特急対応かつ在庫不足の組み合わせ）

## 5. ステップ演習4: データ入力規則（目安25分）

1. 「受注データ」シートに新しい行（15行目）を追加する
2. B15セルを選択し、「データ」タブ→「データの入力規則」を開く
3. 「入力値の種類」を「リスト」に変更し、「元の値」に `=商品リスト`
   （ステップ演習3で作成した名前付き範囲）を指定してOKを押す
4. B15セルにドロップダウンの矢印が表示され、「Tシャツ」「パーカー」「キャップ」の
   3択からしか選べないことを確認する（一覧にない文字を直接入力しようとするとエラーになる）
5. D15セル（数量）に、「データの入力規則」で「入力値の種類: ユーザー設定」を選び、
   次の数式を入力する
   ```
   =AND(D15=INT(D15),D15>=1)
   ```
   → 1以上の整数以外を入力するとエラーメッセージが出ることを確認する
6. エラーメッセージのタブで、タイトル「入力エラー」・エラーメッセージ
   「1以上の整数を入力してください」を設定し、実際に「0」や「1.5」を入力して
   エラーが出ることを確認する

## 8. 応用課題3（時間が余った場合、取り組んでください・目安1.5〜2時間）

### シナリオ

経理担当から「部署・費目ごとの予算上限テーブル」と「経費精算の申請一覧」のデータが
届きました。申請ごとに承認ルートを自動判定し、確認が必要な申請にはフラグを立てる
一覧表を作成してください。

使用ファイル: `day3_extra2_budget_table.csv`（予算上限テーブル: 行=費目、列=部署
（営業部／開発部／総務部）ごとの申請1件あたりの上限額）、
`day3_extra2_expense_requests.csv`（申請データ: 申請ID・部署・費目・金額）

1. 2つのCSVをそれぞれ別シートに取り込む
2. 申請データに「上限額」列を追加し、INDEX＋MATCHで費目×部署の交点から
   上限額を求める数式を入れる（IFERRORでエラー処理も行うこと）
   - 1件だけ、予算上限テーブルに存在しない費目（会議費）の申請が混ざっている。
     エラーにならず「費目要確認」等の分かりやすい表示になるようにすること
3. 「承認ルート」列を追加し、IFSで次の3段階を判定する（上から順に判定されることに注意）
   - 申請金額が上限額を超える → 「部長承認」
   - 申請金額が上限額の50%を超える（かつ上限額以下） → 「上長承認」
   - それ以外 → 「自動承認」
   （IFSの最後の条件には`TRUE`を使い、それ以外すべてに当てはまる条件として指定してよい）
4. 「要確認」列を追加し、「費目が接待費」または「申請金額が上限額を超えている」
   場合に「要確認」と表示する数式を入れる（OR関数を使うこと）
5. 費目の入力欄（新しく申請を1件追加する行を用意する）に、名前付き範囲とデータ入力規則を
   使って、予算上限テーブルに存在する費目からのみ選択できるようにする
6. 「要確認」の申請だけを条件付き書式でハイライトし、フィルターで絞り込めるように
   テーブル機能に変換する
