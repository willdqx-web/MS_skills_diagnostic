## 1. 関数・機能の解説

### 1-1. VLOOKUP関数とは

**「ある表の中から、指定した値と一致する行を探し、その行の別の列の値を取り出す」関数**です。

例えば、社員データの表には「部署コード」（D01, D02, D03…）しか入っていないとします。
このコードだけでは「どこの部署か」が人間には分かりにくいので、別に用意した「部署マスタ」
（部署コードと部署名の対応表）を参照して、部署コードから部署名を自動的に表示させたい、
というときに使います。

#### 書式

```
=VLOOKUP(検索値, 範囲, 列番号, 検索方法)
```

| 引数 | 意味 |
|------|------|
| 検索値 | 探したい値が入っているセル（例: 部署コードが入っているセル） |
| 範囲 | 検索値を探しに行く表全体（**表の一番左端の列に、検索値と同じ種類のデータが入っている必要がある**） |
| 列番号 | 「範囲」の中で、何列目のデータを取り出すか（範囲の一番左の列を「1」として数える） |
| 検索方法 | `FALSE`（または`0`）＝完全に一致するものだけを探す。`TRUE`（または省略）＝近似一致。**基本的には`FALSE`を使う** |

例: `=VLOOKUP(C2, 部署マスタ!$A:$D, 2, FALSE)`
→ C2セルの値（部署コード）を、「部署マスタ」シートのA列〜D列の中からA列で探し、
見つかった行の2列目（部署名）を返す。

#### なぜ範囲を絶対参照（$記号）にするのか

この数式を下の行にコピーしていくと、「検索値」（C2, C3, C4…）は行ごとに変わってほしい
一方、「範囲」（部署マスタの表全体）は常に同じ場所を指してほしいはずです。範囲を
`$A:$D`のように`$`を付けた絶対参照にしておくと、コピーしても範囲がずれません。

### 1-2. XLOOKUP関数（参考・任意）

Excelの比較的新しいバージョンでは`XLOOKUP`という関数も使えます。VLOOKUPより
書き方がシンプルで、列番号を数える必要がありません。

```
=XLOOKUP(検索値, 検索範囲, 戻り範囲)
```

例: `=XLOOKUP(C2, 部署マスタ!$A:$A, 部署マスタ!$B:$B)`
→ 部署マスタのA列からC2と一致する行を探し、その行のB列の値を返す。

使えるバージョンのExcelであれば、VLOOKUPの代わりにこちらを使っても構いません
（今回はどちらか一方ができていればOKです）。

### 1-3. ピボットテーブルとは

**表形式のデータを、行・列・値の3つの軸に自由に組み替えて集計できる機能**です。

例えば「社員ID・部署・月・残業時間」が入った30行のデータがあったとき、
「部署ごとの残業時間合計を知りたい」「さらに月ごとの内訳も見たい」といった集計を、
関数を書かずにフィールドをドラッグするだけで作れます。

#### どんなときに使うと便利か

- データの行数が多く、SUMIFを何度も書くのが大変なとき
- 「部署別」「月別」のように、見たい切り口（軸）が複数あり、
  組み合わせて集計したい（クロス集計）とき
- 集計の切り口を後から変えたい（部署別で見た後、担当者別でも見てみたい等）とき
  → 行・列に配置するフィールドを入れ替えるだけで、数式を書き直さずに集計し直せる
- 元データが更新されたとき → ピボットテーブルを「更新」するだけで最新の集計に反映できる

#### 基本の考え方

ピボットテーブルには4つのエリアがあります。

| エリア | 役割 |
|--------|------|
| 行 | 集計表の縦方向の見出しにしたい項目（例: 部署） |
| 列 | 集計表の横方向の見出しにしたい項目（例: 月） |
| 値 | 実際に集計したい数値（例: 残業時間の合計） |
| フィルター | 特定の条件だけに絞り込みたいとき（今回は使わなくてよい） |

### 1-4. 条件付き書式（復習）

指定した条件を満たすセルだけ、自動的に色や書式を変える機能です。今回の応用課題でも
「残業時間が20時間を超える社員をハイライトする」ために使います
（Day1の実技診断で使った機能と同じです）。

## 2. ステップ演習1: VLOOKUPを使ってみる（目安45分）

使用ファイル: `day2_department_master.csv`、`day2_employee_vlookup_sample.csv`

1. Excelを開き、新しい空白のブックを作成する
2. シート見出し（タブ）を右クリック→「名前の変更」で、シート名を「部署マスタ」に変更する
3. `day2_department_master.csv` をエクスプローラーからダブルクリックして開く
4. 開いたCSVの内容（A1:D4）を選択してコピーし（Ctrl+C）、手順2で作った
   「部署マスタ」シートのA1セルに貼り付ける（Ctrl+V）
5. 「部署マスタ」シートの右側に新しいシートを追加し（シート見出しの「＋」をクリック）、
   シート名を「社員データ」に変更する
6. `day2_employee_vlookup_sample.csv` を開き、内容（A1:E13）をコピーして、
   「社員データ」シートのA1セルに貼り付ける
7. 「社員データ」シートのF1セルに「部署名」と入力する
8. F2セルに次の数式を入力する（Enterで確定）
   ```
   =VLOOKUP(C2,部署マスタ!$A:$D,2,FALSE)
   ```
   → 「営業部」と表示されればOK（C2は「D01」、部署マスタの2列目＝部署名を取得している）
9. G1セルに「課長名」と入力し、G2セルに次の数式を入力する
   ```
   =VLOOKUP(C2,部署マスタ!$A:$D,3,FALSE)
   ```
   → 「佐々木 健一」と表示されればOK
10. H1セルに「内線番号」と入力し、H2セルに次の数式を入力する
    ```
    =VLOOKUP(C2,部署マスタ!$A:$D,4,FALSE)
    ```
    → 「101」と表示されればOK
11. F2:H2の3セルを選択してコピーし、F3:H13（12行目まで）に貼り付ける
    → 部署コードに応じて、それぞれ正しい部署名・課長名・内線番号が表示されることを確認する
    （範囲を`$`で固定してあるため、コピーしても部署マスタの参照先はずれない）
12.（任意）F2・G2・H2の数式を`XLOOKUP`で書き直し、同じ結果になることを確認する
    ```
    F2: =XLOOKUP(C2,部署マスタ!$A:$A,部署マスタ!$B:$B)
    G2: =XLOOKUP(C2,部署マスタ!$A:$A,部署マスタ!$C:$C)
    H2: =XLOOKUP(C2,部署マスタ!$A:$A,部署マスタ!$D:$D)
    ```
    → F2は「営業部」、G2は「佐々木 健一」、H2は「101」と、VLOOKUPのときと
    まったく同じ値が表示されればOK。F2:H2をコピーしてF3:H13に貼り付け、
    全行で正しい値になることも確認する。

#### 付随情報: XLOOKUPで書くとVLOOKUPと何が変わるか

| 項目 | VLOOKUP | XLOOKUP |
|------|---------|---------|
| 取り出す列の指定方法 | 「範囲の左から何列目か」を**数字**で数える（列番号） | 「取り出したい列そのもの」を**範囲として直接指定**する（列番号を数えなくてよい） |
| 検索する列の位置 | 範囲の**一番左端の列**でしか検索できない | 検索範囲と戻り範囲を別々に指定できるため、**検索したい列が表の左端でなくてもよい** |
| 完全一致/近似一致の指定 | 第4引数（`FALSE`/`TRUE`）で毎回指定する必要がある | 既定で完全一致になるため、通常は指定しなくてよい |
| 列を1列増やしたとき | 挿入した列より右側にあるVLOOKUPの列番号がすべてずれ、修正が必要になることがある | 列そのものを指定しているため、表に列を挿入してもずれにくい |

今回のように部署マスタの検索対象（部署コード）が表の一番左端にある場合は、
どちらの関数でも同じ結果になります。違いが実感しやすいのは「検索したい列が
表の右側や真ん中にある」ケースです。今回はどちらの書き方でも結果が同じになることを
確認できれば十分です。

## 3. ステップ演習2: ピボットテーブルを使ってみる（目安45分）

使用ファイル: `day2_attendance_extended.csv`（社員ID・氏名・部署・月・出勤日数・残業時間が
30行分入っています）

1. 新しいシートを追加し、シート名を「勤怠データ」に変更する
2. `day2_attendance_extended.csv` を開き、内容（A1:F31）をコピーして、
   「勤怠データ」シートのA1セルに貼り付ける
3. データ内の任意のセル（例: A1）を選択した状態で、「挿入」タブ→「ピボットテーブル」を
   クリックする
4. 表示されるダイアログでデータ範囲が自動選択されていることを確認し、
   「新規ワークシート」を選んで「OK」をクリックする
   → 新しいシートに空のピボットテーブルが作成される
5. 画面右側の「ピボットテーブルのフィールド」一覧から「部署」を「行」エリアにドラッグする
6. 「残業時間」を「値」エリアにドラッグする
   → 「合計/残業時間」という列ができ、部署ごとの残業時間合計が表示されればOK
7. 「月」を「列」エリアにドラッグする
   → 部署（行）×月（列）のクロス集計表になることを確認する（下記の答え合わせ参照）
8. 「値」エリアの「合計/残業時間」をクリック→「値フィールドの設定」を開き、
   集計方法を「合計」から「平均」に変更してみる
   → 表示される数値が変わることを確認する（確認後、集計方法は「合計」に戻しておく）
9. 「行」エリアに「氏名」を追加する（「部署」の下に置く）
   → 部署の中に社員名が階層表示され、社員ごとの内訳も見られるようになることを確認する
10. ピボットテーブル内の任意のセルを選択した状態で「挿入」タブ→「ピボットグラフ」を
    クリックし、「集合縦棒」を選んでグラフを作成する

|------|-----|-----|-----|

| 開発部 | 68 | 80 | 93 |
