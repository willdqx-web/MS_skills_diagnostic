# Day8 Excelマクロ演習（追加教材・基礎応用編）

> **注記**: これは正式カリキュラムの `08_day8_planner_exercise.md`（Planner実践演習）とは別内容の**追加教材**です。
> Day7の応用課題（社内アンケート集計マクロ）が難しいという声があったため、Day6のステップ演習とDay7の
> 応用課題の間を橋渡しする位置づけで、もう少し易しいレベルのExcelマクロ課題を4問用意しました。
> 実施の要否・タイミング（Day7の前に挟む／Day7の代わりに使う等）は別途判断してください。

## この教材のねらい

Day7の応用課題が難しく感じられた理由として、以下が一度に組み合わさっていたことが考えられます。

- CSVの取り込み（Power Query）
- 複数条件での集計（`AverageIf`を3部署分）
- グラフの自動生成（`AddChart2`・`SetSourceData`）
- 2つの異なる件数のデータでの動作確認
- ボタンへの統合

この教材では、Day7の核心である**「セルを参照するだけでなく、そこから計算をする」「参照するセル・行数を固定値ではなく動的に取得する」**という2点だけに絞り、CSVの取り込みやグラフ作成は扱いません。問題1→問題2→問題3→問題4の順に、組み合わせる要素を少しずつ増やします。

- 問題1: ループで**1行ずつ計算**し、結果を隣のセルに書き込む＋合計・平均を関数で出す
- 問題2: 問題1に**条件分岐（If）**を組み合わせ、行ごとに「該当するかどうか」を判定する
- 問題3: 「動的に参照する」対象を**セル範囲からファイルパスに広げ**、別ファイルを開いて集計する
  （絶対パス・相対パスの違いを体験する）
- 問題4: Day4・Day5で作ったような**ピボットテーブルをマクロから更新し、集計結果を読み取って
  判定する**（ピボットテーブル自体をVBAでゼロから作ることはしません。あくまで
  「更新」と「値の読み取り」に絞ります）

いずれも、Day6のステップ演習7（`lastRow`によるデータ範囲の相対取得）ができていれば取り組める難易度です。
問題4のみ、事前にDay4のピボットテーブル作成（手動操作）ができていることが前提になります。

## 問題1: 売上データの小計・合計・平均を計算するマクロ（目安1〜1.5時間）

### シナリオ

備品の発注担当として、商品ごとの数量・単価が入った一覧表から、行ごとの小計（数量×単価）と、
全体の合計金額・平均単価を自動計算するマクロを作成してください。

### 使用ファイル

`day8_sales_data.csv`（商品名・数量・単価の一覧、8件）

### 準備

1. 新しい空白のブックを作成し、シート名を「売上集計」に変更する
2. `day8_sales_data.csv`の内容をA1セルに貼り付ける（A列=商品名、B列=数量、C列=単価）

### 課題内容

1. VBAエディタで標準モジュールを追加し、`Sub 小計計算()`を作成する
2. Day6のステップ演習7で使った書き方で、データの最終行を取得する
   ```vba
   Dim lastRow As Long
   lastRow = Cells(Rows.Count, 1).End(xlUp).Row
   ```
3. D1セルに見出し「小計」を入れる
4. `For`ループを使い、2行目から`lastRow`まで、各行の**数量×単価**を計算してD列に書き込む
   （`Cells(i, 2)`が数量、`Cells(i, 3)`が単価、`Cells(i, 4)`が書き込み先になる）
   - 固定の行数（例: `For i = 2 To 9`）ではなく、必ず`lastRow`を使うこと
5. ループが終わったあと、`WorksheetFunction.Sum`でD列（小計）の合計を計算し、F1セルに
   見出し「合計金額」、F2セルに計算結果を書き込む
6. 同様に`WorksheetFunction.Average`でC列（単価）の平均を計算し、G1セルに見出し
   「平均単価」、G2セルに計算結果を書き込む
7. マクロをボタンに割り当て、クリックして正しく動くことを確認する
8. **動作確認**: 表に商品を2〜3行追加（またはいずれかの行を削除）してから、同じボタンを
   もう一度押し、コードを書き換えなくても新しい行数に対応した小計・合計・平均が
   正しく再計算されることを確認する

### 確認用の答え合わせ（元の8件データ）

| 商品名 | 小計 |
|--------|------|
| ノートPC | 255,000 |
| マウス | 15,000 |
| プリンター | 44,000 |

合計金額: 467,600円　平均単価: 17,350円

## 問題2: 在庫データの発注要否を判定するマクロ（目安1〜1.5時間）

### シナリオ

在庫担当として、商品ごとの現在庫数と発注点（この数を下回ったら発注が必要になる基準値）の
一覧表から、発注が必要な商品を自動で判定し、件数も集計するマクロを作成してください。

### 使用ファイル

`day8_inventory_data.csv`（商品名・現在庫数・発注点の一覧、8件）

### 準備

1. 新しいシートを追加し、シート名を「在庫チェック」に変更する
2. `day8_inventory_data.csv`の内容をA1セルに貼り付ける（A列=商品名、B列=現在庫数、C列=発注点）

### 課題内容

1. 新しいSub `発注チェック()`を作成する
2. 問題1と同じ考え方で、データの最終行を`lastRow`として取得する
3. D1セルに見出し「発注要否」を入れる
4. `For`ループで2行目から`lastRow`まで、各行について**現在庫数が発注点を下回っているか**を
   `If`文で判定し、
   - 下回っていれば`Cells(i, 4)`に「要発注」と書き込む
   - 下回っていなければ空欄（`""`）にする
   （`Cells(i, 2)`が現在庫数、`Cells(i, 3)`が発注点になる）
5. ループが終わったあと、`WorksheetFunction.CountIf`を使ってD列の中の「要発注」の件数を

6. マクロをボタンに割り当て、クリックして正しく動くことを確認する
7. **動作確認**: 表のいずれかの行の現在庫数・発注点の数値を書き換え、または行を追加・削除して
   から同じボタンを押し、コードを書き換えなくても正しく再判定されることを確認する

## 問題3: 別ファイルの経費データを読み込んで合計するマクロ（目安1.5〜2時間）

### シナリオ

経理担当として、毎月「経費データ.xlsx」という別ファイルで経費の明細が送られてきます。
このファイルを自分の集計用マクロブックと同じフォルダに置き、VBAで自動的に開いて
金額の合計を取得し、閉じるマクロを作成してください。

**今回のポイント**: 問題1・2では「セル範囲」を固定値ではなく`lastRow`で動的に取得しました。
問題3では、**「ファイルの場所（パス）」**についても同じ考え方が必要になることを体験します。
自分のPCのフルパス（絶対パス）をそのまま書いてしまうと、フォルダを移動したり、
別のPCで開いたりした瞬間に動かなくなります。そこで、**自分のブック（マクロを書いている
ファイル自身）の場所を基準にした相対パス**で参照する書き方を学びます。

### 使用ファイル

- `day8_expense_data.xlsx`（経費データ: 日付・費目・金額の一覧、7件が1シートに入っている）

### 準備

1. `day8_expense_data.xlsx`を、これから作る自分のマクロブック（.xlsm）と
   **同じフォルダ**に置く（例: どちらも「マクロ演習」というフォルダの直下に置く）
2. 新しい空白のブックを作成し、シート名を「経費集計」に変更して保存する
   （このブックがマクロを書く本体になる）

### 課題内容（絶対パス版）

1. まず、Windowsのエクスプローラーで`day8_expense_data.xlsx`を右クリック→
   「プロパティ」からファイルの保存場所（フォルダのパス）を確認する
   （例: `C:\Users\自分の名前\Desktop\マクロ演習\day8_expense_data.xlsx`）
2. 新しいSub `絶対パスで読み込み()`を作成し、次のように**フルパスを直接文字列で
   書いて**、経費データのブックを開く
   ```vba
   Dim wb As Workbook
   Set wb = Workbooks.Open("（さきほど確認したフルパス）\day8_expense_data.xlsx")
   ```
3. 開いた別ブックのC列（金額）の合計を`WorksheetFunction.Sum`で計算し、
   `MsgBox`で表示する（`wb.Sheets(1).Range(...)`のように、`wb`を付けて
   「どのブックのシートか」を明示すること）
4. 合計を表示できたら、`wb.Close SaveChanges:=False`で開いた別ブックを閉じる
5. 実行して、経費データの合計（39,300円になるはず）がメッセージで表示されることを確認する
6. **絶対パスの弱点を体験する**: マクロブックと`day8_expense_data.xlsx`が入ったフォルダ
   ごと、別の場所（例えばデスクトップから「ドキュメント」フォルダ内など）に丸ごと移動する。
   そのままマクロを再実行し、**フルパスが変わってしまいエラーになる**ことを確認する
   （「ファイルが見つかりません」といった趣旨のエラーが出る）

### 課題内容（相対パス版）

7. 新しいSub `相対パスで読み込み()`を作成し、今度はフルパスを直接書く代わりに、
   **自分のマクロブック自身がある場所**を基準にパスを組み立てる
   ```vba
   Dim filePath As String
   filePath = ThisWorkbook.Path & "\day8_expense_data.xlsx"

   Dim wb As Workbook
   Set wb = Workbooks.Open(filePath)
   ```
   `ThisWorkbook.Path`は「このマクロが書かれているブック自身が、今どのフォルダに
   保存されているか」を実行のたびに調べてくれる。フォルダの場所が変わっても、
   マクロブックと`day8_expense_data.xlsx`が**同じフォルダに一緒に入っている限り**、
   このパスは自動的に正しい場所を指す
8. 絶対パス版と同様に金額の合計を計算し、`MsgBox`で表示してからブックを閉じる処理を書く
9. **動作確認**: 手順6と同じように、2つのファイルが入ったフォルダごと別の場所へ移動し、
   同じマクロを実行する。今度はコードを一切書き換えなくても、正しく合計（39,300円）が
   表示されることを確認する（これが絶対パスとの違い）

### エラー処理（仕上げ）

10. `day8_expense_data.xlsx`が同じフォルダに存在しない場合に備え、`Dir`関数でファイルの
    存在を確認してから開く処理を追加する
    ```vba
    If Dir(filePath) = "" Then
        MsgBox "経費データファイルが見つかりません: " & filePath
        Exit Sub
    End If
    ```
    このチェックを`Workbooks.Open`より前に入れ、ファイルをわざと別の場所に移動した
    状態で実行し、エラーで止まらずに分かりやすいメッセージが出ることを確認する

### 確認用の答え合わせ

経費データ7件の金額合計: **39,300円**

## 問題4: ピボットテーブルをマクロで更新し、目標達成を判定する（目安1.5〜2時間）

### シナリオ

Day4で作ったような「担当者別の売上ピボットテーブル」を、月ごとに追加される売上データに
合わせて更新する場面を想定します。データが増えるたびに手動で「すべて更新」ボタンを
押す代わりに、**ボタン1つでピボットテーブルを更新し、売上合計が目標額に達しているかを
自動判定する**マクロを作成してください。

**今回のポイント**: ピボットテーブルを**VBAで新規作成することはしません**（それはかなり
高度な内容になります）。ここでの「動的に参照する」は次の2か所だけです。

- ピボットテーブルの名前をコードに固定値で書かず、シート上に**存在するものすべて**を
  対象にする
- ピボットテーブルの総計セルの位置をセル番地で決め打ちせず、`GetPivotData`で**値の
  意味（項目名）から**探す（データが増減すると総計セルの位置がずれるため、
  `Range("F10")`のように固定してしまうと、Day7の応用課題で見つかった「セル範囲を
  固定してしまう」のと同じ失敗につながります）

### 使用ファイル

`day8_pivot_sales_data.csv`（日付・担当者・商品・金額の売上データ、8件）

### 準備

1. 新しいシートに`day8_pivot_sales_data.csv`の内容を貼り付ける（シート名は「売上データ」等）
2. Day4のステップ演習の手順を思い出しながら、このデータをもとに新しいシートに
   ピボットテーブルを1つ作成する（「担当者」を行、「金額」を値に配置するだけでよい）
3. ピボットテーブルの名前を確認しておく（ピボットテーブル内の任意のセルを選択し、
   「ピボットテーブル分析」タブの左端に表示される名前。既定では「ピボットテーブル1」等）

### 課題内容

1. 新しいSub `ピボット更新チェック()`を作成する
2. `For Each`を使い、ピボットテーブルがあるシート上の**すべてのピボットテーブル**を
   更新する（ピボットテーブル名を`"ピボットテーブル1"`のように固定値で書かないこと）
   ```vba
   Dim pt As PivotTable
   For Each pt In ActiveSheet.PivotTables
       pt.PivotCache.Refresh
   Next pt
   ```
3. 更新した後、`GetPivotData`を使って「金額」の総計を取得する
   （項目名から値を探すため、ピボットテーブルの行数が変わってもコードの書き換えは不要）
   ```vba
   Dim total As Double
   total = ActiveSheet.PivotTables(1).GetPivotData("金額").Value
   ```
   （更新は手順2ですべてのピボットテーブルに対して行い、値の読み取りは
   `PivotTables(1)`（シート上の1つ目のピボットテーブル）のように対象を明示すると、
   「どのピボットテーブルの合計を読んでいるか」がはっきりして分かりやすい）
4. 取得した`total`が目標額（例: 450,000円）を超えているかどうかを`If`文で判定し、
   `MsgBox`で「目標達成（合計＝◯◯円）」または「未達成（合計＝◯◯円）」のように、
   判定結果と実際の合計額の両方を表示する
5. マクロをボタンに割り当てる
6. **動作確認**: 元データ（「売上データ」シート）に売上を1〜2行追加してから、
   ピボットテーブルの範囲を確認・拡張し（必要であれば「データソースの変更」で
   範囲を広げ直す）、同じボタンを押して、更新後の合計額で正しく判定されることを確認する

### 確認用の答え合わせ（元の8件データ）

担当者別の売上合計: 佐藤=183,600円、鈴木=229,000円、田中=52,000円
**総計: 464,600円**（目標450,000円の場合 → 「目標達成」と判定されるはずです）
